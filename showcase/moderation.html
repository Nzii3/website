Moderation

Currently, I have 2 moderation systems that you can choose from. v1 includes a more robust command usage, slash commands. While v2 uses prefix commands, however, it uses a custom flags system similar to Wick's flag system and uses a better severity system compared to v1. Both versions include a cases system, in which cases are permanently stored in the database for future use until the case is deleted.

Moderation v1 (slash commands)​

Warning
You cannot use this system with normal prefix commands, it will not be modified to meet your needs usage-wise.

Main Commands​

UsageDescription
/warn <member> [duration][reason] [custom_reason][moderator_hidden]Warn a member (moderator_hidden hides the moderator from the actioned user)
/warnings [member]View warnings for yourself or a member
/remove_warning <id> [notify_user=True]Remove a warning (notify_user will direct message the warned member that it was deleted)
/delete_all_warningsDelete ALL warnings from your server. This is useful if your bot is acting slow when warning users
/ban <member> [reason][custom_reason] [moderator_hidden][delete_message_days]Ban a member from the server (delete_message_days is the amount of message days to delete)
/kick <member> [reason][custom_reason] [moderator_hidden]Kick a member from the server
/purge <amount>Purge an amount of messages in the current channel
/mute <member> <duration> [reason][custom_reason] [moderator_hidden]Mute a member for a duration using Discord's Timeout Feature. (Example duration argument: '1h' = 1 hour)

Warnings that are set to have a duration are removed after the duration, but the case linked to that warning ID is not deleted for further reference.

Cases​

CommandDescription
/case <case_id>View/manage a case - you can edit a case in this command
/createcase <member> <reason> [action=Other][proof=image url] [duration]Create a case for a user
/case [member]View cases for the whole server or yourself

Moderation v2 (prefix commands)​

UsageDescription
!warn <member> <flags>🚩Issue an warning
!case <warning_id>View an warning by ID
!delwarn <warning_id>Delete an warning by ID (Admins only)
🚩 Command Flags

Warning Flags

?s - How sensitive or how bad the warn is for the user; accepts: minor low medium and high as types
?r - The reason for the action, can be any type of string

Mute Flags

?r - The reason for the action, can be any type of string
?t - A time argument for the action (e.g. 1h, 1d, 10m; 1 hour, 1 day, 10 minutes)

Kick Flags

?r - The reason for the action, can be any type of string

Ban Flags

?r - The reason for the action, can be any type of string
?ddays - The number of days to delete messages from the member

Warnings that are set to have a duration are removed after the duration, but the case linked to that warning ID is not deleted for further reference.
